# Thesis radar — beliefstack demo

A minimal end-to-end walk through the five layers, using synthetic
organizational thesis snippets as L0 evidence. Each theme is a revisable
thesis a team might track over time (e.g. *"AI-assisted development is
speeding delivery"*); the outcome label is the strength of supporting
evidence observed in a window — `high` / `moderate` / `quiet`.

## What this demonstrates

| Layer | In this demo |
|---|---|
| L0 events | ~80 invented thesis snippets across 8 organizational themes |
| L1 regions | K-means clusters of the snippets (`k=8`) |
| L2 hypotheses | Per-region empirical prior over evidence-strength outcomes (`high` / `moderate` / `quiet`) |
| L3 lifecycle | First-pass `born` events for every region (multi-pass extension below) |
| L4 calibration | Walk-forward 70/30 split; per-region top-1, top-3, Brier vs held-out outcomes |
| Decisions | PROMOTE / MONITOR / RECLUSTER per region, written into `outputs/region_cards.html` |

The synthetic generator deliberately seeds:

- **stable regions** (e.g. *ai_assisted_development*, *open_source_adoption*) → calibrate cleanly → PROMOTE
- **contradicted regions** (e.g. *remote_collaboration*, *data_privacy_caution*) → train prior misses the test slice → MONITOR or RECLUSTER
- **mixed regions** (e.g. *emerging_runtime_patterns*) → no clear modal outcome → MONITOR

That variety is the point: a real region-card report should not look uniform.

## Data

`data/synthetic_events.jsonl` is generated by:

```bash
python examples/thesis_radar/generate_synthetic_data.py
```

Every event is tagged `SYNTHETIC: …` in its text and `"source": "synthetic_generator"`
in metadata. No real companies, products, or teams appear.

## Run

```bash
pip install -e .
python examples/thesis_radar/run_demo.py
```

Then open `examples/thesis_radar/outputs/region_cards.html`.

## Embeddings

The demo uses `get_default_embedder()`, which returns `MockEmbedder` unless
`OPENAI_API_KEY` is set and the `openai` package is installed. Mock embeddings
are deterministic hash-based vectors — fine for showing the architecture run,
weak for genuine semantic clustering. To use real embeddings:

```bash
pip install beliefstack[openai]
export OPENAI_API_KEY=sk-...
python examples/thesis_radar/run_demo.py
```

## Multi-pass lifecycle (the belief-revision view)

`run_demo.py` runs one calibration pass — every region's lifecycle is `born`.
That proves the pipeline works but doesn't show the *belief revision*.

`run_demo_multipass.py` splits the same data into three chronological eras
and runs two calibration passes, so the lifecycle layer compares pass-B
top-1 against pass-A top-1 and emits a real transition:

```
era 1   Feb 1 - Mar 2     baseline      hypotheses born
era 2   Mar 3 - Apr 1     first revision   born -> strengthened / weakened
era 3   Apr 2 - Apr 29    second revision     -> contradicted / promoted / retired
```

Run it:

```bash
python examples/thesis_radar/run_demo_multipass.py
open examples/thesis_radar/outputs/region_cards_multipass.html
```

Each card in the multi-pass report shows the trajectory inline:

```
lifecycle   born -> strengthened   top1 0.50 -> 0.75
```

That delta is the architecture's contribution. A single classifier would just
report `0.75`; the lifecycle layer says "and it improved from 0.50, which is
why this region is now PROMOTE-eligible."

### Threshold note

The multi-pass demo uses lower lifecycle thresholds than the library defaults
(`min_n_active=2`, `delta=0.10`) because this synthetic dataset is tiny —
~3 events per region per era. Production deployments stay at the defaults
(`min_n_active=5`, `delta=0.05`).

## Caveats

- The synthetic distributions are illustrative, not realistic.
- 80 events across 8 regions is tiny. Production-grade clustering needs orders
  of magnitude more.
- The `EmpiricalHypothesisGenerator` is the simplest possible L2 predictor.
  Sequence-conditional, time-decayed, hierarchical, and calibrated-probability
  variants are documented in the essay ["A pattern for evolving
  beliefs"](https://topicspace.ai/writing/a-pattern-for-evolving-beliefs) but
  not included in this prototype.
