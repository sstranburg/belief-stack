# Sensemaking v1.5 — the cross-substrate bridge

**Read this first — it frames what follows.**

This experiment applies the Belief Stack pattern to a **deliberately different substrate**: an AI-ecosystem *markets* corpus (narratives over public companies), not the operational Claude Code session logs that carry the main planning thesis. It exists to test whether the same maintained-state machinery — claims, warrants, lifecycle, calibration — instantiates on a substrate with very different properties.

**What "success" means here is different from the operational line.** This is a calibration study, not a planning-correctness study. The pre-registered success criterion is **near-chance aggregate calibration with measurable regional heterogeneity** — i.e., the aggregate signal is *expected* to sit near chance, and the finding of interest is that calibration varies systematically across regions of the field. That is the intended result, not a disappointment. Do **not** read the aggregate numbers as a failed prediction system or the per-region returns as a trading claim.

**You will see real tickers and backtest-style returns.** They are the *substrate*, not a product. The market data itself is not redistributed here (see the repo-level [`DATA.md`](../../DATA.md)); what's reported are abstracted reads and summary statistics for methodology illustration.

**Relationship to the main thesis.** Sensemaking v1.5 is a *bridge* — evidence that the architecture is substrate-aware and not specific to coding workflows. It is **not** the operational planning result (`belief_stack_v0_3`, `operational_belief_*`) and should not be cited as such. Different methodology, different success criteria, different substrate.

See `SENSEMAKING_V1_5_PRE_REGISTRATION_v0.4.md` and `SENSEMAKING_V1_5_REPORT_v0_4.md` for the locked design and results.
